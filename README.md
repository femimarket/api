# Femi Rust FFI

A cross-platform Rust library providing a unified FFI (Foreign Function Interface) for the Femi AI application. It exposes AI generation endpoints (image, video, audio, text), a local Project Service for XMP metadata management, and ID3 tag parsing to Swift (iOS/macOS), Kotlin/Android, and WebAssembly.

## Architecture

The project is split into two main layers:

1.  **Rust Core (`Rust/`)**: The source of truth. It contains the business logic, network clients, and platform-specific FFI bindings.
2.  **Platform Bindings**:
    *   **Swift**: `Sources/Api/` wraps the `RustFFI.xcframework` with async Swift APIs.
    *   **Kotlin/Android**: Uses JNI to call the compiled `.so` libraries.
    *   **WebAssembly**: Uses `wasm-bindgen` to expose functions to JavaScript/Kotlin.

### Key Directories

*   `Rust/`: Rust source code, `Cargo.toml`, and build scripts.
*   `RustFFI.xcframework/`: Pre-built Apple framework (generated by `build-rust.sh`).
*   `Kmp/api/src/androidMain/jniLibs/`: Pre-built Android native libraries (generated by `build-rust.sh`).
*   `Kmp/api/src/webMain/kotlin/market/femi/api/RustFfiBundle.kt`: Embedded WebAssembly glue code (generated by `build-rust.sh`).
*   `Sources/Api/`: Swift API extensions.
*   `Tests/ApiTests/`: Swift unit/integration tests.

## Features

### 1. AI Generation Endpoints
All endpoints accept `user` and `password` for authentication. They return binary data (images/videos) or text.

*   **Image Generation**:
    *   `z_image_turbo`: Standard image generation.
    *   `flux2_pro`: Flux 2 Pro image generation.
    *   `nano_banana2`: Nano Banana 2 image generation.
    *   `flux2_dev_i2i`: Flux 2 Dev Image-to-Image.
    *   `flux2_klein_i2i`: Flux 2 Klein Image-to-Image (multi-image input).
*   **Video Generation**:
    *   `ltx2_3a2v`: LTX-2 3D-to-Video (Image + Audio + Prompt).
*   **Audio/Text**:
    *   `qwen3_asr_flash`: Automatic Speech Recognition (Lyrics extraction).
    *   `qwen3_6_35b_a3b`: LLM Chat (Qwen 3 6.35B A3B).

**Cancellation**: All native FFI functions accept a `cancel_flag` pointer. Setting the byte at this address to `1` during execution will abort the request and return a fallback asset (image/video/text) immediately.

### 2. Project Service (XMP Metadata)
Rust owns the file system operations and XMP metadata embedding. It manages a "Documents" directory (sandboxed on Apple, app-specific on Android, OPFS on Web).

*   **Save/Load**: Embeds prompt, model, and subject metadata into files using XMP standards.
*   **Like/Rating**: Toggles a 5-star rating in the XMP metadata.
*   **Character Cast / Image Edit**: In-memory state management for current editing context.

### 3. ID3 Parsing
*   `id3_ffi_extract_sylt`: Extracts synchronized lyrics (SYLT frames) from MP3 files into JSON.

## Installation & Build

### Prerequisites
*   **Rust**: `rustup` installed.
*   **Android NDK**: Required for Android targets.
*   **Xcode**: Required for Apple targets (`xcodebuild`).
*   **Node.js**: Required for WebAssembly inlining.
*   **wasm-bindgen-cli**: Installed automatically if missing.

### Build Script
Run the following from the repository root to build all platforms:

```bash
./build-rust.sh
```

This script produces:
1.  `RustFFI.xcframework`: For Swift Package Manager (iOS 15+, macOS 12+).
2.  `Kmp/api/src/androidMain/jniLibs/*`: For Android `System.loadLibrary("rust_ffi")`.
3.  `pkg/`: WebAssembly output (inlined into JS).
4.  `Kmp/api/src/webMain/kotlin/market/femi/api/RustFfiBundle.kt`: Embedded WASM bundle for Kotlin Multiplatform.

### Manual Build Steps

#### Apple
```bash
cd Rust
export IPHONEOS_DEPLOYMENT_TARGET=14.0
export MACOSX_DEPLOYMENT_TARGET=11.0
rustup target add aarch64-apple-ios aarch64-apple-ios-sim x86_64-apple-ios aarch64-apple-darwin x86_64-apple-darwin
cargo build --release --target aarch64-apple-ios
cargo build --release --target aarch64-apple-darwin
# ... (lipo and xcodebuild commands as seen in build-rust.sh)
```

#### Android
```bash
cd Rust
rustup target add aarch64-linux-android armv7-linux-androideabi x86_64-linux-android i686-linux-android
cargo build --release --target aarch64-linux-android
# ... (copy .so to jniLibs)
```

#### WebAssembly
```bash
cd Rust
rustup target add wasm32-unknown-unknown
cargo build --release --target wasm32-unknown-unknown
wasm-bindgen target/wasm32-unknown-unknown/release/rust_ffi.wasm --out-dir pkg --target web
# Run the inline-wasm node script to embed .wasm into JS
```

## Usage

### Swift (iOS/macOS)
Import the `Api` package. Use the `Api` enum for generation and `ProjectService` for file management.

```swift
import Api

// Generate Image
let imageData = await Api.flux2Pro(
    user: "myuser",
    password: "mypass",
    prompt: "a cat in space"
)

// Save with Metadata
ProjectService.saveFile(
    imageData,
    named: "cat.png",
    prompt: "a cat in space",
    model: "flux-2-pro",
    subject: ["cat", "space"]
)

// Read Metadata
let prompt = ProjectService.getPrompt("cat.png")
```

### Kotlin (Android/JVM)
The Android build produces `.so` files. Load them via `System.loadLibrary("rust_ffi")`. The JNI functions are mapped to Kotlin `external` functions.

```kotlin
// Example JNI mapping (conceptual, based on Rust src/api/jni_api.rs)
class FemiApiJvm {
    init { System.loadLibrary("rust_ffi") }
    
    external fun rustFfiFlux2Pro(user: String, password: String, prompt: String, cancelFlag: Long): ByteArray
}
```

### Kotlin (Web/WASM)
The `RustFfiBundle.kt` file contains the base64-encoded WASM module. Initialize it at runtime.

```kotlin
// Kotlin/WASM
import market.femi.api.RustFfiBundle

// Initialize the WASM module
RustFfiBundle.init()

// Call API
val result = RustFfiBundle.wasmFlux2Pro(user, password, prompt)
```

## API Reference

### Native FFI Functions (`Rust/include/RustFFI/RustFFI.h`)

#### AI Endpoints
All return `uint8_t*` (heap-allocated bytes) and set `size_t* out_len`. The caller must free the pointer using `free()`.

*   `rust_ffi_z_image_turbo(user, password, prompt, cancel_flag, out_len)`
*   `rust_ffi_flux2_pro(user, password, prompt, cancel_flag, out_len)`
*   `rust_ffi_flux2_dev_i2i(user, password, image_b64, prompt, cancel_flag, out_len)`
*   `rust_ffi_flux2_klein_i2i(user, password, image_b64, image2_b64, prompt, cancel_flag, out_len)`
*   `rust_ffi_ltx2_3a2v(user, password, image_b64, audio_b64, prompt, cancel_flag, out_len)`
*   `rust_ffi_nano_banana2(user, password, prompt, cancel_flag, out_len)`
*   `rust_ffi_qwen3_asr_flash(user, password, audio_b64, cancel_flag, out_len)`
*   `rust_ffi_qwen3_6_35b_a3b(user, password, messages_json, cancel_flag, out_len)`

#### Project Service
*   `psxmp_save_file(name, bytes, len, prompt, model, subject, subject_count)`: Saves file with XMP metadata.
*   `psxmp_save_audio(name, bytes, len)`: Saves audio, deleting previous audio files.
*   `psxmp_like(file, liked)`: Sets rating (1=liked, 0=unliked).
*   `psxmp_get_all_generations(buf, buf_len)`: Returns JSON array of filenames.
*   `psxmp_get_audio(buf, buf_len)`: Returns name of current audio file.
*   `psxmp_get_prompt(file, buf, buf_len)`: Returns prompt string.
*   `psxmp_get_model(file, buf, buf_len)`: Returns model string.
*   `psxmp_get_subject(file, buf, buf_len)`: Returns JSON array of subjects.
*   `psxmp_get_like(file)`: Returns 1 if liked, 0 otherwise.
*   `psxmp_read_property(file, ns, name, buf, buf_len)`: Reads arbitrary XMP property.
*   `psxmp_get_url(file, buf, buf_len)`: Returns full file path.
*   `psxmp_set_character_cast(a, b)`: Sets in-memory character cast.
*   `psxmp_get_character_cast(buf, buf_len)`: Returns JSON `[a, b]`.
*   `psxmp_clear_character_cast()`: Clears character cast.
*   `psxmp_set_image_edit(file)`: Sets in-memory image edit target.
*   `psxmp_get_image_edit(buf, buf_len)`: Returns filename.
*   `psxmp_clear_image_edit()`: Clears image edit target.

#### ID3
*   `id3_ffi_extract_sylt(bytes, bytes_len, out_len)`: Returns JSON string of lyrics.

## Non-Obvious Conventions

1.  **Cancellation**: The `cancel_flag` is a pointer to a single `uint8_t`. The Rust side polls this address every 10ms. Setting it to non-zero triggers a graceful abort.
2.  **Memory Management**:
    *   **Native**: Functions returning `uint8_t*` allocate memory via `Box::into_raw`. The caller **must** call `free()` on the returned pointer.
    *   **Swift**: The Swift wrappers use `Data(bytesNoCopy:count:deallocator:.free)` to automatically free the memory when the `Data` object is deallocated.
    *   **Android**: The JNI layer converts raw pointers to `jbyteArray` and frees the Rust memory immediately after copying.
3.  **XMP Metadata**:
    *   Prompt is stored in `dc:description` and `iptc4xmpext:AIPromptInformation`.
    *   Model is stored in `xmp:CreatorTool` and `iptc4xmpext:AISystemUsed`.
    *   Subjects are stored in `dc:subject` array.
    *   Rating is stored in `xmp:Rating` (1-5 for liked, 0 for unliked).
4.  **WebAssembly Inlining**: The WASM module is embedded as base64 into the JavaScript glue code, which is then embedded as a Kotlin string constant. This allows the WASM module to be shipped entirely within the Kotlin library without external assets.
5.  **Fallback Assets**: If a request fails or is cancelled, the Rust FFI returns embedded fallback assets (images/videos) instead of empty data. These are compiled into the binary via `include_bytes!`.
