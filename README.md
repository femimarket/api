# Femi Rust FFI

A cross-platform Rust library providing FFI bindings for the Femi API. It exposes image generation, video generation, audio transcription, and LLM chat endpoints to **iOS/macOS** (via Swift), **Android** (via Kotlin/JNI), and **Web** (via WebAssembly).

It also includes a `ProjectService` module that manages local file storage and XMP metadata embedding, abstracting away platform-specific file I/O (Sandbox on Apple, `filesDir` on Android, OPFS on Web).

## Architecture

The project is split into two main layers:

1.  **Rust Core (`Rust/`)**: The source of truth. Contains the HTTP clients, business logic, and FFI bindings.
2.  **Platform Wrappers**:
    *   **Apple**: `Sources/Api/` (Swift) wraps `RustFFI.xcframework`.
    *   **Android**: `Kmp/api/src/androidMain/` (Kotlin) loads `librust_ffi.so` via JNI.
    *   **Web**: `Kmp/api/src/webMain/` (Kotlin/Wasm) uses an embedded base64 JS bundle generated from `wasm-bindgen`.

### Key Directories

*   `Rust/`: Rust crate source.
    *   `src/api/`: Endpoint implementations (`flux2_pro.rs`, `z_image_turbo.rs`, etc.).
    *   `src/project_service/`: Local file/XMP management.
    *   `src/id3/`: ID3 SYLT (lyrics) extraction.
    *   `include/RustFFI/RustFFI.h`: The C header defining the public API.
*   `Sources/Api/`: Swift extensions for `Api` enum and `ProjectService`.
*   `RustFFI.xcframework`: Pre-built Apple binary (generated by build script).
*   `Kmp/`: Kotlin Multiplatform module consuming the Rust artifacts.

## Building and Running

### Prerequisites

*   **Rust**: `rustup` installed.
*   **Android NDK**: Required for Android targets.
*   **wasm-bindgen CLI**: Installed automatically if missing.
*   **Node.js**: Required for the WASM inlining step.
*   **Xcode**: Required for `xcodebuild` (Apple targets).

### Build Script

Run the main build script from the repository root:

```bash
./build-rust.sh
```

This script performs the following:

1.  **Installs Targets**: Adds `aarch64-apple-ios`, `aarch64-apple-ios-sim`, `aarch64-apple-darwin`, `wasm32-unknown-unknown`, and Android ARM/x86 targets via `rustup`.
2.  **Builds Apple**: Compiles static libraries and merges them into `RustFFI.xcframework`.
3.  **Builds Android**: Compiles `.so` files and places them in `Kmp/api/src/androidMain/jniLibs/<abi>/`.
4.  **Builds WebAssembly**:
    *   Compiles to `wasm32-unknown-unknown`.
    *   Runs `wasm-bindgen`.
    *   **Inlines the `.wasm` binary** into the generated JS glue code as base64.
    *   **Embeds the JS glue code** into a Kotlin constant (`RustFfiBundle.kt`) so it ships inside the KMP library without external assets.

## API Reference

### 1. API Endpoints

All API endpoints follow a similar pattern: they accept credentials, input data (prompts, images, audio), and an optional cancellation flag. They return raw bytes (image/video data or JSON strings).

**Cancellation**: Pass a pointer to a `u8` flag. If the flag is non-zero, the Rust side will abort the request and return a fallback asset (image/video/text) immediately.

#### Image Generation

*   `rust_ffi_z_image_turbo(user, password, prompt, cancel_flag, out_len)`
*   `rust_ffi_flux2_pro(user, password, prompt, cancel_flag, out_len)`
*   `rust_ffi_nano_banana2(user, password, prompt, cancel_flag, out_len)`
*   `rust_ffi_flux2_dev_i2i(user, password, image_b64, prompt, cancel_flag, out_len)`
*   `rust_ffi_flux2_klein_i2i(user, password, image_b64, image2_b64, prompt, cancel_flag, out_len)`

#### Video Generation

*   `rust_ffi_ltx2_3a2v(user, password, image_b64, audio_b64, prompt, cancel_flag, out_len)`

#### Audio & Text

*   `rust_ffi_qwen3_asr_flash(user, password, audio_b64, cancel_flag, out_len)`
    *   Returns lyrics text.
*   `rust_ffi_qwen3_6_35b_a3b(user, password, messages_json, cancel_flag, out_len)`
    *   `messages_json`: JSON array of `[{role, content}]`.
    *   Returns updated JSON array with the assistant's reply appended.

### 2. Project Service (Local Storage & XMP)

Rust owns the Documents directory. All functions take a **filename** (not a path). Rust resolves the full path internally.

*   `psxmp_save_file(name, bytes, len, prompt, model, subjects, count)`: Saves a file and embeds XMP metadata.
*   `psxmp_save_audio(name, bytes, len)`: Saves audio, deleting any existing audio files.
*   `psxmp_like(file, liked)`: Sets XMP Rating to 5 (liked) or 0 (unliked).
*   `psxmp_get_all_generations(buf, buf_len)`: Returns JSON array of filenames.
*   `psxmp_get_audio(buf, buf_len)`: Returns the name of the current audio file.
*   `psxmp_get_prompt(file, buf, buf_len)`: Returns prompt string.
*   `psxmp_get_model(file, buf, buf_len)`: Returns model string.
*   `psxmp_get_subject(file, buf, buf_len)`: Returns JSON array of subjects.
*   `psxmp_get_like(file)`: Returns 1 if liked, 0 otherwise.
*   `psxmp_get_url(file, buf, buf_len)`: Returns the full file path.
*   `psxmp_set_character_cast(a, b)`: Sets in-memory character cast pair.
*   `psxmp_get_character_cast(buf, buf_len)`: Returns JSON `[a, b]`.
*   `psxmp_clear_character_cast()`: Clears in-memory cast.
*   `psxmp_set_image_edit(file)`: Sets in-memory image edit target.
*   `psxmp_get_image_edit(buf, buf_len)`: Returns filename.
*   `psxmp_clear_image_edit()`: Clears in-memory edit target.

### 3. ID3 SYLT Extraction

*   `id3_ffi_extract_sylt(bytes, bytes_len, out_len)`: Extracts synchronized lyrics from MP3 bytes. Returns JSON array of timed lines.

## Usage Examples

### Swift (iOS/macOS)

```swift
import Api

// Generate Image
let imageData = await Api.flux2Pro(
    user: "myuser",
    password: "mypass",
    prompt: "a cat in space"
)

// Save with Metadata
ProjectService.saveFile(
    imageData,
    named: "cat.png",
    prompt: "a cat in space",
    model: "flux-2-pro"
)

// Retrieve Metadata
let prompt = ProjectService.getPrompt("cat.png")
```

### Kotlin (Android/JVM)

```kotlin
import market.femi.api.FemiApiJvm

// Generate Image
val bytes = FemiApiJvm.rustFfiFlux2Pro(
    user = "myuser",
    password = "mypass",
    prompt = "a cat in space",
    cancelFlag = 0L
)

// Project Service
ProjectServiceJvm.psxmpInitDocuments(context.filesDir.absolutePath)
ProjectServiceJvm.psxmpSaveFile("cat.png", bytes, "a cat", "flux-2-pro", null)
```

### Kotlin (Web/Wasm)

```kotlin
import market.femi.api.RustFfi

// Initialize once
RustFfi.init()

// Generate Image
val bytes = RustFfi.flux2Pro(
    user = "myuser",
    password = "mypass",
    prompt = "a cat in space"
)

// Project Service (OPFS)
RustFfi.psxmpSaveFile("cat.png", bytes, "a cat", "flux-2-pro", emptyList())
```

## Non-Obvious Conventions

1.  **Memory Management**:
    *   FFI functions return `*mut u8` and set `out_len`.
    *   The **caller** is responsible for freeing the memory (using `free()` in C/Swift, or `Box::from_raw` in Rust tests).
    *   Kotlin wrappers handle this automatically by converting to `ByteArray`.
2.  **Cancellation**:
    *   The `cancel_flag` is a pointer to a single `u8`.
    *   The Rust side polls this flag every 10ms.
    *   Setting it to non-zero aborts the HTTP request and returns a fallback asset.
3.  **WebAssembly Inlining**:
    *   The WASM binary is not served as a separate file. It is embedded as base64 in the JS glue code, which is itself embedded as a Kotlin string constant.
    *   This allows the KMP library to be fully self-contained without npm dependencies or asset bundling.
4.  **XMP Metadata**:
    *   Apple uses `xmp_toolkit` (smart handlers).
    *   Android/Web use `xmpkit_body` (pure Rust bytes manipulation).
    *   Both support `dc:description`, `iptc4xmpext:AIPromptInformation`, `xmp:CreatorTool`, `iptc4xmpext:AISystemUsed`, and `dc:subject`.
